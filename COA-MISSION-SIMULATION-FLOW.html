Sentinel OS — COA → Mission Planning → Simulation Pipeline

Dokumentasi komprehensif alur data end-to-end dari Intelligence Feed, Course of Action (COA) generation, Mission Planning, hingga Tactical Simulation dalam platform Sentinel OS. Mencakup arsitektur frontend, backend agentic AI, custom type system, dan data flow di setiap stage.

Sentinel OS adalah platform kesadaran situasional (situational awareness) berbasis web yang menyatukan berbagai sumber informasi ke dalam satu layar operasional. Platform ini dirancang multi-sektor — bisa digunakan untuk pertahanan & keamanan, kebencanaan (BNPB), penegakan hukum, oil & gas, logistik, hingga konservasi lingkungan.

Detail setiap step:

Step 1: Gather Terrain (Mapbox)

File: server/services/mapbox-service.js → gatherTerrainContext()

Mengambil data dari Mapbox API:

  • Elevation grid (SRTM data)
  • Reverse geocode (place name)
  • Nearby facilities (airports, ports, hospitals via POI)
  • Isochrone zones (15/30/60 min driving range)
  • Driving routes dari facilities terdekat ke lokasi event
  • Satellite + topographic imagery (Base64 untuk vision input ke LLM)
  • Water/land detection
  • Weather conditions (Open-Meteo: suhu, angin, visibility, curah hujan, flood risk, air quality, marine conditions)

Step 2: Build User Prompt

File: server/services/prompts.js → buildUserPrompt()

Menggabungkan:

  • Intel event data (title, summary, category, threat level, location, tags)
  • Elevation data summary
  • Geocode data
  • Nearby facilities dengan koordinat
  • Approach routes dengan jarak/durasi
  • Satellite + topographic images (multimodal vision input)
  • Previous result (jika regeneration/revision)
  • User revision request
  • SOP content
  • Custom analysis prompt

Step 3+: LLM Tool-Calling Loop

File: server/services/openai-tools.js → callOpenAIWithTools()

Agentic loop — LLM secara otonom memutuskan tools mana yang perlu dipanggil:

Tool File Handler Fungsi
query_knowledge_repository knowledge-repository.js Cari berita/intel terbaru dari knowledge repo API
get_terrain_context mapbox-service.js Dapatkan data terrain untuk lokasi baru
get_weather_conditions mapbox-service.js + Open-Meteo Weather real-time + flood risk + marine
recommend_naval_assets asset-recommender.js Rekomendasi asset TNI AL (multi-criteria scoring + sub-agent)
search_ais_vessels ais-service.js Cari kapal di database AIS berdasarkan nama/MMSI
get_vessel_track ais-service.js Track history 24 jam AIS untuk vessel tertentu
get_vessels_in_area ais-service.js Semua kapal di area maritim (by sea name atau bbox)

Config:

  • Model: configurable via LLM_MODEL env
  • Max tokens: 16,384
  • Max tool rounds: 5
  • Max retries: 3
  • Parallel tool calls: enabled

Setiap tool call dikirim ke frontend via SSE sebagai event: step.

Config:

  • Model: configurable via LLM_MODEL env
  • Max tokens: 16,384
  • Max tool rounds: 20 (lebih banyak dari COA karena planning lebih kompleks)
  • Max retries: 10 (lebih resilient)
  • Parallel tool calls: enabled
  • Reasoning effort: medium

Key differences dari COA Agent:

  • Mission Planner menambahkan calculate_routes tool (Mapbox Directions API)
  • Tool calls di-log ke database (agentToolCalls table) untuk audit
  • Reasoning steps di-log ke database (agentReasoning table)
  • Output di-transform ke TacticalPlan format yang sangat terstruktur

LLM Output Structure

LLM menghasilkan JSON besar dengan:

  1. Mission Overview — name, type, status, description
  2. Operational Details — commander, comms channel, ROE
  3. AOI — center coords, radius, bounds, key locations
  4. Objectives — primary, secondary, constraints
  5. Threat Assessment — level, score, factors (with weights)
  6. Execution Phases — id, name, status, description
  7. Tactical Brief — phases with unit assignments (callsign, mission, route, ETA, modules)
  8. Map Annotations — routes, symbols, areas dengan geo-coordinates
  9. Route Options — distance, duration, difficulty, waypoints
  10. Risk Mitigation — primary strategy + contingencies

3.4 Backend — Simulation Agent

File: server/services/simulation-agent.js
Function: generateSimulation()

Pipeline:

graph TD
    S1["Step 1: Analyze Plan\n(buildPlanSummary)"] --> S2["Step 2: LLM Call\n(Single-shot, no tool loop)"]
    S2 --> S3["Step 3: Validate and Enrich\n(enrichSimConfig)"]
    S3 --> S4["Step 4: Save to DB"]
    S4 --> S5["Step 5: Return SimConfig"]

Stage 3 berbeda dari COA/Mission: Simulation Agent TIDAK menggunakan tool-calling. Ini adalah single-shot LLM call karena semua data yang dibutuhkan sudah ada di mission plan.

Step 1: Build Plan Summary

File: simulation-agent.js → buildPlanSummary()

Mengekstrak dari TacticalPlan:

  • Mission name, description, threat level
  • AOI center coordinates
  • Commander, comms channel, ROE
  • Execution phases dengan descriptions
  • Unit tasking (callsign, type, mission, route, ETA)
  • Map features (drawings) dengan coordinates
  • Objectives

Step 2: LLM Single-Shot Call

System prompt menginstruksikan LLM untuk generate:

Field Deskripsi
units[] Array unit dengan type, callsign, color, geo-coordinate path
units[].path[] 10-25 waypoints: { t, lng, lat, alt, heading, speed, action }
groundTargets[] Target posisi statis: { id, label, lng, lat, type, destroyTime }
events[] Timeline events: { time, type, label, unitId, score, location }
phases[] Phase timing: { name, startTime, endTime }
aoi Camera setup: { center, zoom, pitch, bearing }
duration Total playback durasi (120-300 detik)

Rules untuk waypoint generation:

  • Aircraft: 200-500 km/h, smooth flight paths
  • Helicopters: 100-250 km/h
  • Ground/Naval: 20-80 km/h
  • Paths mengikuti routes dari mission plan drawings
  • Enemy units approach dari luar AOI
  • 1° ≈ 111 km (digunakan untuk offset calculation)

Config:

  • Model: configurable via LLM_MODEL env
  • Max tokens: 16,384
  • Max retries: 3
  • Temperature: 1 (higher creativity for varied simulations)

Step 3: Enrich SimConfig

File: simulation-agent.js → enrichSimConfig()
  • Assign consistent unit IDs
  • Set default colors: ally #00bfff, enemy #ff2255
  • Normalize unit types (player → ally)
  • Ensure AOI from plan viewport
  • Normalize waypoint defaults (heading, speed, action)
  • Set min duration to 180s if too short

Step 4: Save to Database

File: server/services/db.js → simulationDataStore.create()

Saved sebagai JSON di simulation_data table.

3.5 Frontend — Map Simulation Playback

Simulation di-render menggunakan dua komponen:

A. MapSimulation (Primary — Mapbox GL)

File: src/components/simulation/MapSimulation.tsx

Render langsung di atas Mapbox GL map yang sudah ada. Menggunakan native Mapbox layers (bukan Canvas overlay):

Layer Type Fungsi
sim-units-src GeoJSON Source Posisi semua unit + ground targets
sim-icons-lyr Symbol Layer Render chevron (air) / square (ground) icons
sim-labels-lyr Symbol Layer Callsign labels
sim-trail-{unitId} Line Layer Movement trail per unit

Geo interpolation:

File: src/types/simulation-types.ts → interpolateGeo()
  • Interpolasi linear antara waypoint berdasarkan waktu t
  • Menghitung heading dari 2 consecutive positions
  • Altitude digunakan untuk flight levels display

SVG Sprites:

  • Chevron (ally blue / enemy red) — untuk air units
  • Square (ally blue / enemy red) — untuk ground/naval units
  • Target crosshair / destroyed target

Animation loop (requestAnimationFrame):

  1. Update GeoJSON source data (satu setData() call)
  2. Update trail LineStrings (setiap 5 frame untuk performance)
  3. Process events (destroyed, detected, phase_change, alert)
  4. Camera follow — smooth pan ke allied unit setiap 2 detik
  5. Advance time berdasarkan speed multiplier

Playback controls:

  • ▶ Play / ⏸ Pause / ⟳ Replay
  • Speed: 1× / 2× / 4×
  • Seek bar (click to jump)
  • 💾 Save / 📥 Export JSON / ✕ Close

SimUnit — Unit Bergerak di Peta

interface SimUnit {
    id: string;
    callsign: string;          // e.g. "EAGLE-1"
    type: 'ally' | 'enemy';    // Menentukan warna (blue vs red)
    unitType: UnitRole;        // Menentukan shape icon
    color: string;             // Hex, e.g. '#00bfff'
    shadowColor: string;       // Glow effect
    
    // GEO-COORDINATE WAYPOINT PATH — ini yang di-animate
    path: SimWaypoint[];
    
    destroyTime?: number;      // Detik kapan unit dihancurkan (null = survive)
}

interface SimWaypoint {
    t: number;     // Detik dari start
    lng: number;   // Longitude (geo-coordinate)
    lat: number;   // Latitude (geo-coordinate)
    alt?: number;  // Altitude meters (untuk flight level display)
    heading?: number; // Degrees
    speed?: number;   // km/h
    action?: 'cruise' | 'loiter' | 'strike' | 'evade' | 'land' | 'hover';
}

Kunci: SimWaypoint.path[] menggunakan real geo-coordinates (lng/lat), BUKAN pixel coordinates. Ini yang membuat simulasi bisa di-render langsung di atas Mapbox GL map.

Geo Interpolation Engine

File: src/types/simulation-types.ts → interpolateGeo()

function interpolateGeo(path: SimWaypoint[], t: number): { lng, lat, alt, heading } | null

Pada setiap animation frame, posisi unit dihitung dengan:

  1. Cari 2 waypoint yang mengapit waktu t saat ini
  2. Hitung fraction f = (t - a.t) / (b.t - a.t)
  3. Terapkan cubic easing untuk smooth motion: ef = f < 0.5 ? 2f² : -1 + (4-2f)f
  4. Interpolasi: lng = a.lng + (b.lng - a.lng) × ef (sama untuk lat, alt)
  5. Auto-calculate heading dari atan2(dLng, dLat) jika tidak disediakan

Key Design Decisions

  1. SSE over WebSocket — Semua AI generation menggunakan Server-Sent Events (unidirectional) karena cukup untuk streaming reasoning steps. WebSocket tidak diperlukan karena tidak ada bidirectional communication selama generation.

  2. Shared Tool System — openai-tools.js menyediakan tool definitions dan handlers yang digunakan bersama oleh COA agent dan Mission Planner agent. Nama file tetap "openai-tools" karena alasan historis, namun mendukung berbagai LLM provider. Ini menghindari duplikasi kode dan memastikan konsistensi.

  3. Single-shot Simulation — Simulation agent tidak menggunakan tool-calling karena semua data (koordinat, unit, fases) sudah ada di mission plan. Ini membuat simulation generation lebih cepat (~5-10 detik vs ~30-60 detik untuk mission planning).

  4. GeoJSON on Mapbox GL — Map simulation menggunakan native Mapbox GL layers (bukan Canvas overlay) untuk GPU-accelerated rendering. Ini memungkinkan smooth animation bahkan dengan banyak unit.

  5. Data Continuity — SOP content dan custom prompt dibawa dari COA stage ke Mission Planning stage melalui React Router navigation state, ensuring the AI has consistent context across the pipeline.

  6. Multi-Session Safety — Setiap agent invocation completely stateless (no shared mutable state). Multiple concurrent users dapat generate COA/plans/simulations secara bersamaan tanpa race conditions.

  7. LLM Provider Abstraction — llm-client.js mendukung berbagai LLM provider sebagai backend. Model configurable via LLM_MODEL env var. Ini memungkinkan switching provider dan model tanpa code changes.

  8. Mock Fallback — Frontend selalu memiliki fallback ke mock data jika backend unreachable. Ini memastikan demo bisa berjalan tanpa API key.
